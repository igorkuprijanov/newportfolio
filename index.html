Delicatesso Restorante

Restournt landing page.

Link to the website Link to source

A landing page for an italian restourant written in pure JavaScript and using Animate.css for the animations. Effectively a single-page website where different page sections are brought up in form of menu cards through use of Animate.css 'slide-up'-style animations.

The page has a 'book a table' popup function where a disired table and time of booking can be selected, also animated with Animate.css. The website is fully responsive and usable on moblie devices.

The website does not use an external API, rather all data is kept in the main html file. This approach has been chosen due to the fact that restourants rearly change their menus, contacts or locations so there is no need for frequent data updates, yet if they are needed, the main file can be easily modified.

Novum Tabernam

Cosmetics company web store.

Link to the website Link to source

This website is a web store for a cosmetics company. Written using React and its JSX syntax extention. As a single page website it uses React Router to navigate through different sections of the store. To display the items in 3D space I used Three.js's React version React Three Fiber.

The website has few categories where the products are displayed, a dynamicly updated shoping cart which also takes the user to the checkout page, and a 'liked' section where favorite products can be bookmarked. For the data there is a simulation of an API, it is just a simple array of objects that is located outside of the main files and can be easily updated and changed without modifying the actual store files.

The models are not of the best quality, but they do however serve their purpose, to display a 3D interactable object of the product.

Spider Solitare Express

Spider solitare card game.

Link to the website Link to source

A classic spider solitare game, that one can play by oneself. Made with pure JavaScript. The game is played with two decks of cards and goes on until all eight foundation piles are filled. Not every game is winnable, that is just the nature of the game, due to cards beeing distributed randomly. The website also has a set of rules and explanation of how to play the game.

The game uses Deck of Cards API to draw card objects with their provided images. For moving the cards I used HTMLs Drag and Drop API, which turned out not as good as it was planned. The API is not supported on mobile devices, thus the game is unplayable on mobile or touch screen devices.

Every stack of cards is an array of objects of these cards. During the game arrays are modified depending on the actions of the player. The foundation piles are a part of an boolean array, when all items of the array are set to the value of true, the game is won and the endgame message is displayed.
